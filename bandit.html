OverTheWire : Bandit

Rapport d'apprentissage : Niveaux 0 à 13 (Manipulation de données)

Qu'est-ce que Bandit ?

Bandit est une succession de challenges techniques (CTF ou Capture The Flag) liée à l'apprentissage de l'environnement Linux et aux fondamentaux de la Cybersécurité.

Les challenges se basent sur un principe progressif, où chaque niveau impose de résoudre une problématique spécifique : recherche et manipulation de fichiers, analyse des données et protocoles réseaux afin d'extraire le mot de passe permettant d'accéder au niveau suivant.

Niveau 0 → 1 : Initialisation SSH

Analyse : Pour ce premier niveau d'initialisation, il nous est simplement demandé de se connecter en SSH au serveur bandit.labs.overthewire.org sous l'utilisateur bandit0 avec le mot de passe bandit0 via le port 2220.

$ ssh bandit0@bandit.labs.overthewire.org -p 2220

Une fois connecté, je fais la commande ls et je m'aperçois de la présence d'un fichier « readme ». Je l'ouvre donc grâce à la commande cat readme qui m'affiche le mot de passe contenu à l'intérieur.

$ ls readme $ cat readme The password you are looking for is: ZjLjTmM6FvvyRnrb2rfNWOZOTa6ip5If

Niveau 1 → 2 : Fichiers Spéciaux (-)

Analyse : Il m'est demandé de lire un fichier appelé - que nous pouvons appeler « dashed filename ». Le problème c'est qu'un fichier nommé de la sorte ne peut pas être simplement lu via une commande cat ou nano ordinaire.

Sur Linux, de nombreuses commandes utilisent le symbole - pour indiquer une option ou autres entrées (stdin).

Solution : La solution est alors d'utiliser le chemin relatif du fichier pour le lire en ajoutant ./ devant le nom du fichier.

$ cat ./- 263JGJPfgU6LtdEvgfWU1XP5yac29mFx

Niveau 2 → 3 : Espaces dans les noms

Analyse : Il m'est indiqué que le mot de passe se trouve dans un fichier appelé spaces in this filename. Comme dit, ce fichier contient des espaces dans son nom. Le problème, c'est qu'on ne peut pas lire simplement un fichier contenant des espaces.

La commande cat ou nano pense que je tente d'entrer une option, un peu à la manière du premier niveau.

Solution : Pour contrer cela, il faut ajouter \ avant chaque espace ou utiliser des guillemets.

$ cat spaces\ in\ this\ filename MNk8KNH3Usiio41PRUEoDFPqfxLPlSmx

Niveau 3 → 4 : Fichiers Cachés

Analyse : La consigne m'indique que le mot de passe est contenu dans un fichier se trouvant dans le dossier inhere. Je me déplace alors dans ce fichier. Mais il n'y a aucun fichier apparent quand j'utilise simplement la commande ls.

Dans ce cas-là, je peux tenter la commande ls -a (all) pour montrer tous les objets présents et même ceux qui sont masqués. On y découvre alors un fichier nommé ...Hiding-From-You. Les fichiers dont le nom commence par un . sont considérés comme cachés, ce qui explique que nous ne le voyions pas.

$ cd inhere/ $ ls -a . .. ...Hiding-From-You $ cat ...Hiding-From-You 2WmrDFRmJIq3IPxneAaMGhapopFhF3NJ

Niveau 4 → 5 : Types de Fichiers (Human-readable)

Analyse : La consigne indique que le mot de passe se trouve dans le seul fichier lisible par un humain. On peut donc en déduire qu'il y a plusieurs fichiers illisibles et qu'un parmi cette liste est lisible.

Effectivement, il y a plusieurs fichiers qui semblent illisibles ou corrompus qui en altèrent même le Shell.

Solution : Pour ne pas s'embêter à lire les fichiers un par un, nous pouvons utiliser la commande file qui nous indique de quel type de fichier il s'agit réellement, en ajoutant l'argument ./* pour analyser tous les fichiers du dossier.

$ file ./* ./-file07: ASCII text # Le fichier -file07 est le seul identifié comme "ASCII text". $ cat ./-file07 40QYVPkxZO0E005pTW81FB8j8lxXGUQW

Niveau 5 → 6 : Recherche avancée (Find)

Analyse : Dans ce niveau, le fichier est caché dans une arborescence de plusieurs sous-dossiers. Manuellement, il est impossible de le retrouver.

La consigne nous donne quelques éléments pour le retrouver : lisible par un humain, son poids est de 1033 octets et il n'est pas exécutable. Dans cette situation, la commande find nous sera utile en y ajoutant ces arguments :

  • -type f : Chercher uniquement des fichiers.
  • -size 1033c : Filtrer précisément sur la taille (c pour octets).
  • ! -executable : Le point d'exclamation signifie « NOT » (exclure les exécutables).
$ find . -type f -size 1033c ! -executable ./maybehere07/.file2 $ cat ./maybehere07/.file2 HWasnphtq9AVKe0dmk45nxy20cvUa6EG

Niveau 6 → 7 : Recherche serveur & Redirections

Analyse : A présent, le fichier n'est plus stocké dans notre répertoire personnel mais il peut être n'importe où sur le serveur. Nous avons quelques indications : le fichier appartient à l'utilisateur bandit7 et au groupe bandit6 et sa taille est de 33 octets.

Solution : Comme au niveau précédent, la commande find va nous aider. J'ajoute 2>/dev/null pour rediriger les erreurs (flux 2) vers « le vide » (null) pour ne voir apparaître que le fichier auquel nous avons accès et qui nous intéresse.

$ find / -user bandit7 -group bandit6 -size 33c 2>/dev/null /var/lib/dpkg/info/bandit7.password $ cat /var/lib/dpkg/info/bandit7.password morbNTDkSW6jIlUc0ymOdMaLn0lFVAaj

Niveau 7 → 8 : Grep

Analyse : La consigne nous indique que le mot de passe est stocké à côté du mot millionth dans le fichier data.txt.

Pour cela, la commande grep va nous être utile. Elle va chercher une ligne dans le fichier qui contient le mot spécifié.

$ grep millionth data.txt millionth dfwvzFQi4mU0wfNbF0e9RoWskMLg7eEc

Niveau 8 → 9 : Uniq et Sort

Analyse : Le mot de passe est stocké dans le fichier data.txt et est la seule chaîne de caractères unique dans le fichier. Cela veut dire que le fichier est probablement rempli de doublons.

Solution : Nous allons utiliser deux commandes. D'abord sort pour trier par ordre alphabétique les lignes. Puis uniq -u qui nous permet de n'afficher que la ligne qui n'apparaît qu'une seule fois.

$ sort data.txt | uniq -u 4CKMh1JI91bUIZZPXDqGanal4xvAg0JM

Niveau 9 → 10 : Strings

Analyse : On nous dit que le mot de passe est stocké dans l'une des rares chaînes de caractères lisibles par l'homme, précédée d'une répétition de plusieurs « = ».

Nous allons donc utiliser strings pour pouvoir lire le fichier (qui n'est pas lisible simplement) et dans la même commande utiliser grep pour chercher une ligne contenant plusieurs « = ».

$ strings data.txt | grep === ========== the password is FGUW5ilLVJrxX9kMYMmlN4MgbpfMiqey

Niveau 10 → 11 : Encodage Base64

Analyse : Le mot de passe est stocké dans le fichier data.txt codé en Base64.

Il nous suffit de le lire grâce à la commande base64 -d (decode).

$ base64 -d data.txt The password is dtR173fZKb0RRsDFSGsg2RWnpNVj3qRr

Niveau 11 → 12 : Chiffrement ROT13

Analyse : Le fichier contient du texte qui semble lisible (caractères ASCII) mais qui ne veut rien dire. Il s'agit en effet d'un chiffrement ROT13 (rotation de 13 caractères). Étant donné que l'alphabet est composé de 26 lettres, il suffit de faire une rotation de 13 de plus pour revenir aux caractères d'origine.

Commande : tr 'A-Za-z' 'N-ZA-Mn-za-m'. On indique que l'on veut décaler les lettres de N à Z dans le périmètre de A à M (et idem pour les minuscules).

$ cat data.txt | tr 'A-Za-z' 'N-ZA-Mn-za-m' The password is 7x16WNeHIi5YkIhWsfFIqoognUTyj9Q4

Niveau 12 → 13 : Compression (La Poupée Russe)

Analyse : Dans ce niveau, nous devons faire preuve de patience et de méthodologie. Le fichier d'origine est un « hexdump » qui cache une succession de compressions dans différents formats (Gzip, Bzip2, Tar). Nous pouvons voir cela un peu comme une poupée Russe : un fichier compressé qui contient un autre fichier compressé et ainsi de suite.

Il nous est indiqué que dans le répertoire personnel nous n'avons pas les droits pour créer des fichiers. Il faut alors créer un dossier dans le /tmp avec la commande mktemp -d.

Procédure :

  1. Premièrement, inverser (décoder) le hexdump grâce à xxd -r (revert) vers le dossier temporaire.
  2. Utiliser la commande file pour savoir de quel type de fichier il s'agit (Gzip, Bzip2, Tar).
  3. Renommer le fichier avec la bonne extension (ex: .gz) puis l'extraire avec la commande appropriée (gunzip, bunzip2, tar -xf).
  4. Répéter l'opération jusqu'à obtenir le fichier en clair.
$ mktemp -d /tmp/tmp.y9p1AJTqks $ xxd -r data.txt > /tmp/tmp.y9p1AJTqks/data.bin # Après de multiples décompressions... $ cat data8 The password is 8ZjyCRiBWFYkneahHsk5WjRARs0P1Zvr

Conclusion

La validation de ces 12 premiers niveaux, axés sur la manipulation de fichiers et l'administration système, a été une étape déterminante.

Ces bases fondamentales m'ont permis d'acquérir une réelle fluidité dans l'environnement Linux, une aisance indispensable pour aborder sereinement les challenges plus complexes à venir.
